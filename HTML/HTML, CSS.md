# Введение в создание Web-приложений

## Как устроено Web-приложение?

Давайте начнем с вопроса: **Что вы представляете себе, когда думаете о Web-приложении?**

Web-приложение состоит из нескольких ключевых частей:

1. **Страница** — то, что вы видите в браузере.
2. **Сервер** — это "за кулисами", то, что обрабатывает ваши запросы (например, регистрацию или вход).
3. **База данных** — хранилище информации (например, список зарегистрированных пользователей).
4. **Протокол или API** — язык, на котором "общаются" страница, сервер и база данных.

![Diagram 2](attachments/Diagram%202.svg)

Представьте, что вы отправляете форму регистрации. Вы видите только страницу, но сервер обрабатывает ваш запрос, а информация сохраняется в базе данных. Это простая схема взаимодействия.

## HTML: основа любой веб-страницы

### Что такое HTML?

Когда мы заходим на страницу, она состоит не только из текста, в ней можно выделить отдельные части: блоки, заголовки,
обычный текст, списки, кнопки, формы. Именно язык HTML помогает «разметить» страницу этими элементами. То есть HTML —
это основа любой веб-страницы.

>**HTML (Hypertext Markup Language)** — это язык разметки, с помощью которого создаются веб-страницы. С его помощью мы можем структурировать информацию на странице: заголовки, текст, списки, кнопки, формы и т.д.

### Основы HTML

HTML строится на **тегах**. Вот основные виды тегов:

- `<tag>` — **открывающий тег**
- `</tag>` — **закрывающий тег**
- `<tag>Контент внутри тега</tag>` — **тело тега**
- `<tag />` — укороченная запись для пустых тегов.

**Пример:**

```html
<h1>Привет, HTML!</h1>
<p>Это абзац текста.</p>
```

HTML-документ образует дерево, которое называется **DOM** (Document Object Model). Элементы страницы как будто "вложены" друг в друга, образуя структуру.

## Важные теги HTML

### Заголовки

Теги **h1**...**h6** используются для создания заголовков разного уровня.

```html
<h1>Заголовок 1 уровня</h1>
<h2>Заголовок 2 уровня</h2>
```

Чем выше уровень заголовка (h1 — h6), тем он меньше по размеру. Попробуйте поиграть с этими тегами и посмотрите на результат!

### Абзацы текста

Тег **p** используется для создания абзацев.

```html
<p>Это пример абзаца текста.</p>
```

### Блочные и строчные элементы

Язык разметки HTML постарался придумать теги на каждый элемент, который нужен на страничке: мы встретимся в дальнейшем с
тегами для картинок, ссылок, меню навигаций и многого другого.

Однако всегда есть элементы, которым нельзя найти подходящий тег. Например, мы хотим объединить наш заголовок и его
описание в единый элемент. Подходящего тега для этого нет, однако на этот случай были придуманы теги **span** и **div**. Их
называют _несемантическими тегами_, а все прочие теги, соответственно, — _семантическими_.

- **div** создает отдельные блоки, размещаемые на новой строке.
- **span** используется для выделения части текста внутри строки.

**Пример использования:**

```html
<div>
    Это отдельный блок.
</div>
<span>Это текст внутри строки.</span>
```

**Строчные теги** встраиваются как часть строки. Например, с помощью тега **span** можно выделить часть текста отдельным
цветом:
![](attachments/Pasted%20image%2020230917202323.png)

```html
<div>
    Купите солнечные батареи <span>прямо сейчас</span> и спасите планету.
</div>

```

**Блочные теги** — это отдельный блок на странице. Когда мы помещаем их внутрь строчки, они переносятся на следующую
строчку:
![](attachments/Pasted%20image%2020230917202437.png)

```html
<div>
    Купите солнечные батареи <div>прямо сейчас</div> и спасите планету.
</div>

```

По возможности, нужно использовать семантические теги, так как это облегчает всяким автоматическим парсерам работу (
например, робот крупных поисковиков типа Яндекса или Гугла, индексирующий вашу страницу, сможет лучше понять, что на ней
находится и отобразить корректную информацию в результатах поиска)

### Ссылки и кнопки

Если вам нужно, чтобы при нажатии на слово пользователь переходил на другую страницу, можно использовать ссылки (
выглядеть это может, например, [так](https://www.youtube.com/watch?v=dQw4w9WgXcQ))

Ссылки создаются с помощью тега **a**:

```html
<a href="https://example.com">Это ссылка</a>
```
- ссылка может быть как абсолютной (начинаться с http:// и вести на какой-то другой сайт), та и относительной (
  например, `register.html`, тогда браузер сам превратит это в `http://*адрес текущей страницы*/register.html`)

Кнопки создаются с помощью тега **button**:

```html
<button onclick="alert('Нажата кнопка!')">Нажми меня</button>
```

Мы еще поговорим о событиях вроде `onclick` на следующих занятиях!

### Картинки

Картинки добавляются с помощью тега **img**:

```html
<img src="https://example.com/image.png" alt="Описание картинки">
```

Обратите внимание: тег **img** не требует закрывающего тега.

### Немного про структуру html файла

Простейший HTML файл будет выглядеть примерно так:

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>
</head>
<body>
<p>Test</p>
</body>
</html>
```

Из всего написанного, мы разобрали только тег `<p>`, зачем нужно всё остальное?

```html
<!DOCTYPE html> - означает, что дальше будет файл на языке HTML
<html lang="en"> - как говорилось выше, html документ - это дерево элементов, а значит должен быть и корень. Это как раз
он
<head> - элементы в теге head не отображаются на странице, обычно тут происходит подгрузка необходимых для корректного
    отображения страницы ресурсов
    <meta charset="UTF-8"> - кодировка файла(если не понимаете, о чем речь - не страшно)
    <title>Title</title> - заголовок (название вкладки, отображаемое вверху в браузере)
</head>
<body> - тут расположены элементы, которые надо отображать (не всегда, но в большинстве случаев)
<p>Test</p>
</body>
</html>
```

## CSS: как сделать страницу красивой

**CSS** (Cascading Style Sheets) — это язык стилей, который изменяет внешний вид элементов на странице.

### Основы

**CSS** — язык, основанный на правилах.

Например, правило: у всех заголовков первого уровня (тег **h1**) будет шрифт **Open Sans** и черный цвет. В правиле можно
выделить две части: сначала _кому_ (тегу **h1**, например), затем _что_ (шрифт **Open Sans**, черный цвет)._Кому_— это
CSS-селекторы, а_что_— это CSS-свойства.

Например, правило «_у всех заголовков первого уровня (тег_**_h1_**_) будет шрифт_**_Open Sans_**_и черный цвет»_выглядит
так:

Пример простого CSS-правила для заголовков:

```css
h1 {
    color: blue;
    font-family: Arial, sans-serif;
}
```

Здесь задается стиль для всех заголовков **h1**: синий цвет и шрифт Arial.

### Селекторы

CSS применяет стили к элементам на основе **селекторов**. Есть три основных типа:

1. **Теги** — применяется ко всем элементам одного типа. Пример: `h1 { ... }`
2. **Классы** — задаются для множества элементов. Пример:

```html
<div class="example"></div>
```

CSS для класса:

```css
.example {
    background-color: yellow;
}
```

3. **ID** — уникальный идентификатор для одного элемента:

```html
<div id="unique"></div>
```

CSS для ID:

```css
#unique {
    background-color: green;
}
```

## Не изобретаем велосипеды: готовые решения

### CodePen

Если вам нужно что-то готовое или вдохновение, можно воспользоваться сайтом [CodePen](https://codepen.io/), где можно найти множество готовых HTML/CSS решений.

### Bootstrap

Если вам не хочется тратить время на создание стилей вручную, используйте **Bootstrap** — набор готовых стилей и компонентов. Чтобы его подключить, добавьте следующие строки в `<head>` вашей страницы:

```html
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.2.3/dist/css/bootstrap.min.css" rel="stylesheet">
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.2.3/dist/js/bootstrap.bundle.min.js"></script>
```

С Bootstrap можно быстро создать профессионально выглядящую страницу без глубоких знаний CSS.
